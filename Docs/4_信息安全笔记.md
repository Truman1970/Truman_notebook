## 计算机硬件基础

### 中央处理器-CPU
CPU基础功能：指令控制、操作控制、时间控制、数据加工。
CPU的组成：运算器、控制器、Cache。

### 控制器-CU
- 功能：指挥计算机各个部件协调一致的自动运行。
- 组成：程序计数器-PC、指令寄存器-IR、指令译码器、时序产生器、操作控制器。

### 运算器-ALU
- 功能：在控制器的指挥下，进行算数运算和逻辑运算。
- 组成：算数逻辑单元-ALU、通用寄存器、数据缓冲寄存器-DR、状态条件寄存器-PSW。

### CPU中主要寄存器
- 数据缓冲寄存器—DR：暂时存放ALU的运算结果、或由数据存储器读出的一个数据字，或来自外部接口的一个数据字。
- 指令寄存器-IR：保存当前正在执行的一条指令。
- 程序计数器-PC：存放下一条指令的地址，确定下一条指令。
- 数据地址寄存器—AR：保存当前CPU所访问的Cache存储器中单元的地址。
- 通用寄存器-R0-R3：当ALU执行算数运算或者逻辑运算时，为ALU提供一个工作区。
- 状态寄存器-PSW：保存由算数指令和逻辑指令运算或测试结果建立的各种条件代码。进位和借位保存在状态寄存器里，根据值判断是否由进位和借位。

> 构成运算器的部件中，最核心的是**算术和逻辑运算单元**


### 指令的寻址方式

指令格式：操作数+地址码

- 隐含寻址，指令中隐含操作数地址，一般来说，操作数存储在专用寄存器中。
- 立即寻址，操作数存储在指令中，不需要访问存储器，速度最快。
- 直接寻址，操作数地址存储在指令中，操作数存储在存储器中。
- 间接寻址，相比于直接寻址，两次寻址才取到操作数。
- 寄存器寻址，寄存器地址存放在指令中，操作数存储在寄存器中。
- 寄存器间接寻址，相比于寄存器寻址，两次寻址才取到操作数，操作数存储在存储器中。
- 偏移寻址，寄存器直接寻址 + 寄存器间接寻址。

> 获取操作数速度最快的是**立即寻址**。
> 操作数地址在某寄存器的寻址方式为**寄存器间接寻址**。

### 存储器

- 一个存储器中所有存储单元的总数称为它的容量。
- 静态存储器很快，断电信息丢失。
- 存储器分级：速度由快到慢，价格由高到低：寄存器、Cache、主存（RAM）、外存。


### 总线和接口

CPU和内存交互**按照传输信息**分类：数据总线、地址总线、控制总线。
- 数据总线
  - 用于传输数据信息。
  - 数据信息在数据总线上双向传输。
  - 数据总线的根数决定了通过该总线一次可以传送的信息量。
- 地址总线
  - 用来传输地址信息，地址信息在地址总线上单向传输。
  - 地址总线的根数决定了总线可以直接寻址的内存地址空间大小。
  - n根地址总线可以寻址的地址空间是2的n次方。
- 控制总线
  - 用于在主控设备和被控设备之间传输控制信号。

CPU和内存交互**按照所处位置**分类
- 片内总线
  - 发生在CPU内部。
  - CPU内部多个功能部件相连的数据传输通路。
  - CPU内部运算器、控制器以及寄存器之间通过片内总线传输数据。
- 系统总线
  - 发生在主机内部。
  - 又称为内总线，是CPU内存和各种接口电路之间的信息通路。
- 通信总线
  - 发生在主机外面。
  - 又称为外总线，是计算机和计算机之间或计算机和其他通信设备之间的信息传输通路，如电缆、网线、光纤等。

**接口**：临时存储信息的部件。
- 并行接口
  - 主要用于连接打印机或者扫描仪。
  - 并行传输受到线路长度限制，只适合10M以内的数据。
  - 并行接口适合高速、短距离通信。
- 串行接口
  - 用来连接Modem或其他串行通信设备。
  - 串行速度较慢，通信距离远，抗干扰强，连线成本低。
- 硬盘接口
  - IDE采用并行方式进行数据通信，接口速度快，价格低，兼容性强，每个IDE接口可以连接两个IDE设备，分别称为主设备和从设备。
  - SATA接口采用串行数据传输方式，可靠性高，抗干扰性强，支持热拔插，每个接口只能连接一个SATA设备，比较主流。
  - SCSI接口数据传输率高，CPU占用率低，除了连接硬盘外，还可以连接光驱、扫描仪、打印机设备等，较为常用。

> 断电后，存储在**RAM**的数据会丢失。  
> 解决主存和CUP之间速度不匹配的部件是**Cache**。  
> 总线宽度分为地址总线宽度和数据总线宽度，其中地址总线宽度决定了CPU能够使用多大容量的**主存储器**。  
> 若地址总线的宽度为32位，则最多允许**2^32b**访问的物理空间(如果没有标准答案，就选最接近的，如4GB，商家拿B代替b)。  

若内存地址区间为4000H~43FFH，每个存储单元可存储16位二进制数，该内存区域用4片存储器构成，则该内存所用的存储器芯片的容量是多少？
```shell
43FFH - 4000H + 1H = 4400H - 4000H = 400H = 2^10D
(2^10 * 2) / 4 = 2^9B
```

某计算机存储空间按字节编址，若某区域的地址空间为4A000H~4DFFFH，则该段内存区域的容量是多少？
```shell
4DFFFH - 4A000H + 1H = 4E000H -4A000H = 4000H = 2^14D
2^14 * 1 = 2^14B
```

思考：8G的内存，装32位的操作系统好？还是装64位的操作系统好？  
解：2^32b = 2^9MB ≈ 4GB，寻址空间只能是4GB。  

### 外部设备
- 输入设备：键盘、鼠标。
- 输出设备：显示器、打印机、绘图仪。
- 外部存储器：磁带、软盘、硬盘、光盘、U盘、移动硬盘。

磁盘存储设备
- 磁道的编址从外往内依次编号，最外的同心圆叫0磁道，最里面的同心圆叫n磁道。
- 每个盘面由许多磁道的同心圆组成。
- 扇区的编号可以是连续的，也可以间隔编号。

磁盘存储设备的技术指标
- 存储密度
  - 道密度，每一个磁盘上有多少磁道。
  - 位密度，单位磁道长度上能够记录二进制的位数，相同磁道位密度相同，0磁道位密度最小。
  - 面密度，整个磁盘面上可以存储多少信息，道密度和位密度的乘积。
- 存储容量
  - 格式化容量，写入文件系统之后的容量。
  - 非格式化容量，一般格式化容量是非格式化容量的60%-70%。
- 平均存取时间
  - 平均存取时间 = 平均找道时间 + 平均等待时间 + 数据传送时间
- 数据传输率
  - 单位时间内向主机传输数据的字节数（字节/秒）。

> 相比于温度、磁场、湿度，对磁盘影响最小的是**噪声**。  
> **错误**：同一个磁盘上每个磁道的**位密度**都是相同的。  
> **正确**：同一个磁盘上每个磁道的**容量**都是相同的。  
> **正确**：0磁道密度最小，n磁道密度密度最大。  
> **正确**：提高磁盘的转速一般不会减少平均寻道时间，会减少等待时间。  
> **正确**：磁盘的格式化容量一般都比非格式化容量小。  

### 冯诺依曼体系
两个思想
- 计算机五大部件：运算器、控制器、存储器、输入设备、输出设备。
- 存储程序原理：把要执行的指令和待处理的数据，按照顺序存储到计算机的内部存储器中，程序和数据以二进制的形式不加区别的存放，存放的位置由内存地址决定。

- 内存寻址空间，和地址总线根数相关。
- 存储周期

## 计算机网络基础
### 计算机网络类别
- 按照不同作用范围
  - 广域网WAN，几十到几千公里，也成为远程网。
  - 城域网MAN，五到五十公里，一个城市或者几个大单位。
  - 局域网LAN，一公里左右，单位、公司、学校等。
  - 个人区域网PAN，家用。
- 按照不同使用者
  - 公用网，电信建立的公用网。
  - 专用网，军队、铁路、电路。
- 用来把用户接入到英特网的网络，ISP运营商，又叫本地接入网。


### 常见网络拓扑结构
- 总线型
  - 特点：通过一条电缆连接所有节点，一般支持30台计算机，过多计算机额外需要中继器。
  - 特点：半双工工作，一个时刻只能由一台计算机传输数据。
  - 优点：架设成本低。
  - 优点：容易安装。
  - 缺点：故障后果严重，任意一个节点出现故障，整个网络瘫痪。
  - 缺点：故障诊断困难。
  - 缺点：传输效率较低。
- 星型
  - 特点：中间有一个集线器（以前）或者交换机（现在），所有计算机通过中间的集线设备访问其他计算机。
  - 优点：故障诊断容易。
  - 优点：网络稳定性较好，故障隔离简单，一旦某一台计算机发生故障，不会影响整个网络。
  - 优点：可扩展型强。
  - 优点：提供网络传输速率容易。
  - 缺点：费用高，主要体现在中间的集线设备。
  - 缺点：布线麻烦。
  - 缺点：对中间节点依赖度较高。
- 树型
  - 特点：局域网常见的是树型结构，从根节点往下看，分别是核心层、汇聚层、接入层，公司常用。
  - 优点：易于网络扩展。
  - 缺点：故障诊断困难。
  - 缺点：对根设备依赖较高。
- 环型
  - 适用于大型网络。
  - 优点：网络可用性高。
  - 缺点：故障诊断难。
- 网状型
  - 缺点：线路成本高，控制复杂。
  - 优点：可靠性高，稳定性好。

> 单位网络拓扑结构一般严格保密，提高攻击难度。  
> 如果采用总线型网络拓扑结构，该网络传输信息的方式是**随机争用**。   
> 错误：**总线型拓扑结构**最大的缺点是：一旦中心节点故障，整个网络瘫痪。  
> 正确：在**令牌环网络拓扑结构**中，要发送信息的终端必须要获得令牌状态为0的令牌才能发送消息，并且在发送消息时将令牌状态置换为1。   

### OSI的体系结构
- 第一层，物理层。
  - 作用：透明传输**比特流**。处理信号通过介质的传输。
  - 主要设备：中继器（加强衰减信号，提高传输距离）、集线器（简单转发比特流，不进行碰撞检测）。
- 第二层，数据链路层。
  - 作用：透明传输**帧**信息。帧传输的时候会校验数据的顺序和完整性，保证数据正确的顺序、无差错和完整性。
  - 主要设备：网桥（接收、存储帧信息，查找转发表，然后传输帧信息）、二层交换机（连接设备、扩充网络范围）、网卡。
  - 常见协议：PPTP（单隧道协议）、L2TP（多隧道协议）、SLIP（串行线路网际协议）、PPP（点对点协议）。
- 第三层，网络层。
  - 作用：分组传输和路由选择。控制报文通过网络时的路由选择。
  - 主要设备：三层交换机（有路由功能）、路由器（通过路由表，实现网络通信）。
  - 常见协议：ARP（利用IP地址找MAC地址）、RARP（知道MAC地址找IP地址）、IP、ICMP（网际控制报文协议）、IGMP（网际组管理协议）。
- 第四层：传输层
  - 作用：端到端的连接。保证端到端的可靠性传输。
  - 常见协议：TCP（三次握手、可靠连接）、UDP（不可靠连接）。
- 第五层：会话层。
  - 作用：建立、管理和终止会话。
  - 常见协议：POP3、FTP、TFTP、HTTP、Telent、SMTP、SNMP、DHCP、DNS。
- 第六层：表示层。
  - 作用：数据的格式与表达、加密、压缩。
  - 常见协议：POP3、FTP、TFTP、HTTP、Telent、SMTP、SNMP、DHCP、DNS。
- 第七层：应用层。
  - 作用：为用户的应用进程提供服务。提供用户和网络的接口。
  - 常见协议：POP3、FTP、TFTP、HTTP、Telent、SMTP、SNMP、DHCP、DNS。

> 网桥的特点：使用网桥连接的网络在数据链路层以上采用相同或者兼容的协议。  
> 网桥的特点：使用网桥可以隔离冲突域。  
> 网桥的特点：使用网桥可以接收、存储、地址过滤及转发数据。  
> 错误：网桥处理接收到的数据**降低了**时延，提高了网络性能。  
> 正确：网桥处理接收到的数据**增加了**时延，因为多了碰撞检测。  

### TCP/IP体系结构（现在常用）
- 第一层，网络接口层，由物理层和数据链路层组成。
- 第二层，网际层（网络层）。
  - 常见协议：IP、ICMP（报告网络情况）、OSPF（路由选择协议）、ARP、RARP。
- 第三层，传输层。
  - 常见协议：TCP、UDP。
- 第四层，应用层，由会话层、表示层和应用层组成。
  - 常见协议（通过TCP传输）：HTTP、FTP、SMTP、Telnet。
  - 常见协议（通过UDP传输）：DNS、RIP、SNMP、DHCP。

## 对称密钥密码模型
密码的安全需求
- confidentiality 保密性
- integrity 完整性
- availability 可用性

古典密码学要解决的问题：通信保密

三个古典密码系统
- 羊皮传书，两个一样的棍子
- 藏头诗，我爱秋香，又叫隐写术
- 凯撒密码，已知最早的替换密码

相关概念
- P: plaintext 明文
- C: ciphertext 密文
- K: key 密钥
- E: encipher(encrypt) 加密(加密算法)
- D: decipher(decrypt) 解密(解密算法)

这五要素构成一个密码系统，被称为对称密钥密码模型
```
E: P + K >>> C 
D: C + K >>> P 
```
